# 心流工作區

作為白板系統的延伸工具，心流工作區提供了一個免干擾的執行環境。它允許使用者從白板匯入大方向的任務，在沙盒內進行拆解、計時、筆記與執行，最終將執行結果打包匯出，過程完全不污染原有的白板資料庫。
白板網址:https://progress01.github.io/workboard/ 
工作區網址:https://progress01.github.io/workflow/

## 🛠 系統定位與架構

* **沙盒機制 (Sandbox Model)**：資料獨立儲存於本機端的 `flow_workspace_data`，任何任務的新增、修改、拆解與刪除，皆不會直接影響母體白板的狀態。
* **白板連動 (Integration)**：
* **無縫登入**：共用白板系統的 Firebase Google 驗證 (Auth)。
* **單向匯入**：可直接讀取白板系統的 LocalStorage 暫存，將「排程卡片」與「發想節點」轉換為待辦任務，並自動根據顏色或標籤指派優先級 (P1/P2/P3)。


* **前端技術**：純 HTML, CSS (CSS Variables, Flexbox 排版), Vanilla JavaScript。無套件依賴。

## ✨ 核心功能模組

### 1. 執行控制面板 (左側面板)

* **⏱️ 計時器系統**：
* **彈性計時 (Flexible)**：正向計時，適合無壓力的長時間沉浸。
* **番茄時鐘 (Pomodoro)**：標準 25 分鐘倒數機制。
* **死線倒數 (Deadline)**：可設定特定專案的截止日期，系統會自動計算剩餘天數並以不同警戒色標示。


* **📋 任務矩陣 (Todo List)**：
* 支援快速新增任務並指派優先權 (P1 紅 / P2 黃 / P3 藍)。
* 自動標示任務來源（白板匯入、發想轉移、合併、分割）。
* **任務合併 (Merge)**：可勾選多個零碎任務，一鍵合併為單一任務，並自動串接原始筆記內容。



### 2. 動態焦點筆記區 (右側面板)

* **📝 雙模式編輯器**：
* 未選取任務時：作為「全域沙盒草稿」使用。
* 點擊任務「聚焦」時：編輯器會自動綁定該任務，所寫的筆記與修改的標題皆會跟隨該單一任務。


* **⚡ 編輯輔助工具**：
* 支援攔截 `Tab` 鍵進行 4 空格縮排，方便層級化紀錄。
* 內建 Markdown 語法快捷鍵 (粗體、斜體、程式碼、清單)。
* 內建四大情境模板：`🐛 Bug`、`🔍 拆解`、`📝 覆盤`、`✂️ 分隔線`。


* **✂️ 任務拆解功能 (Split)**：
* 針對過大的任務，可在筆記中使用 `---` 作為斷點，點擊「拆解」即可將大任務自動切割成多個獨立的子任務。


* **閃念紀錄 (Quick Log)**：
* 輸入短語按 Enter，系統會自動打上時間戳記 (Timestamp) 並將其附加至當前筆記的最下方。



### 3. 結算與報告匯出

系統內建高度自動化的 Markdown 報告產生器 (`generateReportMarkdown`)，可將一日的戰果進行統整：

* **自動計算戰報統計**：包含已完成、剩餘任務數與完工率 (%)。
* **層級降噪排版**：依照 P1、P2、P3 進行任務分類，並將任務內的筆記自動進行 Markdown 縮排，確保閱讀體驗的整潔。
* **匯出方式**：
1. **複製報告**：一鍵複製格式化的 Markdown 純文字，可直接貼回「白板」的全域筆記本進行存檔。
2. **下載 .md**：生成帶有當日日期的 `.md` 實體檔案儲存至本機。
3. **🧹 清空沙盒**：結算完畢後，提供一鍵安全清空沙盒資料的功能，迎接下一個工作日。
